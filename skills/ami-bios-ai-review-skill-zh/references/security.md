# 安全分支

这个分支产出 `安全发现`。

重点关注漏洞、信任边界失守、不安全的特权行为、不安全的构建流程以及二进制来源问题。

纯风格问题如果没有遮蔽真实安全风险，可以忽略。

## 输出规则

每条 finding 应说明：

- 文件类型或文件族
- 安全风险类别
- 缺陷点
- 固件影响
- 联动复核目标

## 按文件类型的安全规则

### C / H / CPP

重点检查：

- 指针有效性，以及可能导致整数溢出的尺寸计算
- 缓冲区边界、复制长度、结构体打包、类型转换安全性
- MMIO / I/O 位宽是否正确、posted write 顺序、寄存器副作用
- 认证、信任检查、权限假设
- 状态传播、失败清理、部分初始化的硬件状态
- SMM、PEI、DXE、MM 边界错误

联动检查：

- `INF`、`DEC`、`DSC`、`DEPEX`、寄存器定义、库绑定

### ASM / S / NASM / ASM16 / CSM16 / INC / EQU / MAC / NASMB

重点检查：

- 寄存器保存 / 恢复是否正确
- 栈纪律和调用约定是否符合
- 中断屏蔽与恢复行为
- 模式切换假设与段寄存器状态
- 临时 RAM、复位向量、trampoline 的信任假设
- 是否把敏感寄存器 / 内存暴露在外

联动检查：

- 复位向量配置、相邻 C 包装层、阶段入口、平台说明

### INF / DEC / DSC / DEPEX

重点检查：

- 模块类型是否错误地扩大了权限或运行范围
- 是否缺少依赖声明，从而隐藏所需安全库或协议
- 是否绑定了错误的库，削弱了安全路径
- 特性开关是否暴露了被禁用或未签名的路径
- 依赖表达式是否允许模块在安全保护建立前执行

联动检查：

- 源码文件、`SDL`、`CIF`、平台 `DSC`、Capsule 与 Secure Boot 模块

### SDL / CIF / MAK

重点检查：

- Token 默认值是否暗中启用 Debug 或不安全策略
- eLink / 组件顺序是否让代码在保护措施建立前插入执行
- 变体 / 组件包含是否意外扩大攻击面
- 构建开关是否改变签名、Capsule、Boot Guard、TPM 行为

联动检查：

- `INF`、`DSC`、`SSP`、安全库、二进制打包规则

### SD / HFR / VFR / UNI / SSP

重点检查：

- Setup 选项是否把敏感安全开关暴露给了错误用户或错误场景
- 变量绑定错误是否让 UI 控件写错字段
- 默认值是否削弱平台策略
- 字符串文本是否错误描述了真实安全状态
- NVRAM / CMOS 定义是否重叠、泄露状态或绕过完整性检查

联动检查：

- Setup 数据结构、callback 代码、`UNI`、`SSP`、`SDL`、NVRAM 服务

### ASL / ASLC / DXS

重点检查：

- ACPI Method 是否暴露敏感设备状态或写入危险值
- AML 生成源是否过早 / 过宽地发布设备或能力
- 依赖源是否在所需协议尚未安全建立前激活模块
- TPM、电源和固件接口方法是否信任未验证输入

联动检查：

- `DEPEX`、驱动、协议生产者、平台策略模块

### 脚本与工具

适用于：

- `PY`、`BAT`、`SH`、`CMD`、`GAWK`

重点检查：

- 不安全命令执行
- 路径注入、通配符误用、环境假设
- 在签名 / 镜像生成路径中调用了未签名或未验证工具
- 吞错行为是否可能生成部分产物或被篡改产物
- 密钥处理、口令材料与敏感信息泄露

联动检查：

- 签名资产、生成二进制、打包元数据、发布流程文档

### 配置 / 数据表

适用于：

- `YAML`、`JSON`、`XML`、`MXML`、`INI`、`CNF`、`DEF`、`DAT`、`DT`、`ASI`、`MCB`、`CBIN`、`TXT`

重点检查：

- 安全敏感默认值
- 是否解锁保护区域、削弱 strap / fuse / policy
- 微码或 FIT 表是否不匹配
- 纯数据改动是否带来隐藏的安全策略变化
- 文本形式 FDF / reserve 区声明是否移动受信任组件

联动检查：

- 消费代码、平台构建元数据、Flash 镜像组成

### 二进制 / 镜像 / 信任资产

适用于：

- `FD`、`FV`、`FFS`、`EFI`、`BIN`、`ROM`、`RAW`、`LIB`、`DLL`、`EXE`、`SO`、`PDB`、`CER`、`PEM`、`KEY`、`ESL`、`SIGSTRUCT`

重点检查：

- 来源与可追溯性
- 是否发生意外二进制替换
- 密钥 / 证书角色是否不匹配
- Debug / 符号产物是否误带入发布
- 镜像组成变化是否绕过签名或验证假设

联动检查：

- 构建脚本、清单、签名流程、`SDL`、`DSC`、FDF 文本声明

## 安全默认倾向

- Secure Boot 和 Boot Guard 流程默认高敏感
- SMM 和 MM 变更默认高敏感
- 密钥材料和签名流水线改动即使不改代码，也默认高敏感
